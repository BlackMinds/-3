## 一、系统概述

### 1.1 设计目标

万界仙途是单角色修仙挂机游戏，目前的成长循环主要围绕**自身境界突破**与**装备/功法养成**展开。玩家在中后期容易形成"个人英雄独行"的孤独感。道侣系统通过引入**情感长线 + 伴生养成 + 子嗣传承**三层叙事，把"修道"从单人变成"道侣"再变成"家族"，达成以下目标：

- **填补社交情感空缺**：宗门系统是异步弱社交，秘境组队是临时强社交，**道侣是唯一的"长期 1v1 强情感绑定"**
- **拉长养成深度**：道侣 → 亲密度 → 生育 → 子女培养 → 子女出战，形成 100+ 小时长线目标
- **强化修仙题材沉浸**：道侣、仙缘、血脉、家族传承，全部是修仙小说核心桥段
- **增加付费深度（可选）**：高品质道侣稀有度天然支撑商业化空间，但**不能 P2W**

### 1.2 系统定位

| 维度 | 定位 |
|------|------|
| 玩法类型 | 长线伴生养成（非战斗主线） |
| 角色属性 | 道侣 = 永久 buff 提供者 + 解锁特殊玩法；子女 = 可出战助战单位 |
| 核心吸引 | 亲密度阶段化解锁、道侣加成、子女血脉觉醒 |
| 与现有系统 | 复用：游历事件、五行灵根、秘境引擎、装备背包；新增：道侣表/亲密度/子女表 |
| 解锁前置 | **金丹期**（境界 ≥ 3）首次开启，与宗门系统同期解锁 |
| 商业化 | 道侣礼物礼包（仙玉）、子女资质重铸丹（极稀有） |

## 二、道侣获取系统

### 2.1 设计原则

- **不抽卡、不商城直购**：道侣获取**完全靠游历事件**，强化"机缘"感
- **品质纯随机**：邂逅时按全局概率随机品质，**与地图等级无关**（避免后期玩家洗高品道侣的刷图行为）
- **非保底**：一次邂逅不一定结侣，需要后续培养亲密度才能正式结侣
- **永久不消失**：邂逅后录入"道侣花名册"，未结侣的也保留（可后续选定）

### 2.2 游历系统（道侣邂逅唯一入口）

> **重要：道侣邂逅完全脱离"打怪历练"机制。** 玩家必须主动选择"游历"才能触发邂逅事件，强化"散修游历红尘求道"的修仙叙事，并彻底消除"高等级刷低级图洗邂逅"的漏洞。

#### 2.2.1 入口与定位

| 项目 | 设定 |
|------|------|
| **UI 入口** | 主界面 → 历练 Tab → 子 Tab "**游历**"（与"普通历练 / 离线挂机 / 秘境组队"并列） |
| **解锁条件** | 境界 ≥ 金丹（tier 3） |
| **核心定位** | 道侣邂逅 + 修仙小奇遇的双轨产出，独立于战斗系统 |
| **战斗系统隔离** | **打怪、闭关、秘境、宗门、塔等所有战斗均不触发邂逅** |

#### 2.2.2 每日游历次数

| 来源 | 次数 |
|------|------|
| 基础（金丹+） | 3 次 / 天 |
| 宗门 5 级 | +1 次 / 天 |
| 七夕活动期间 | +2 次 / 天 |
| 仙玉商城应急（限购） | +1 次（每周限购 2 次，¥6 一次） |
| **每日上限** | **5 次（封顶，无法突破）** |

> 上限 5 次 + 仙玉每周仅 +2，**无法通过氪金无限刷邂逅**。每日 00:00 重置（与现有 `daily-reset` cron 同步）。

#### 2.2.3 游历地点（按境界解锁）

不同地点对应不同灵根偏向的道侣，玩家可按需选择（增加策略性，但**不能定向刷高品质** —— 品质永远按全局概率）：

| 地点 ID | 名称 | 解锁境界 | 灵根偏向 | 风格 |
|---------|------|---------|---------|------|
| `falling_dawn_town` | 落霞古镇 | 金丹 | 金/土 | 烟火人间，偶遇凡间女修 |
| `myth_beast_outskirts` | 万妖山外围 | 元婴 | 火/木 | 妖兽密林，偶遇剑修 |
| `ghost_sea_coast` | 幽冥海岸 | 化神 | 水 | 神秘海港，偶遇水系道侣 |
| `thunder_wasteland` | 天雷荒原 | 渡劫 | 金 | 雷劫之地，偶遇雷修 |
| `void_immortal_realm` | 太虚仙境 | 大乘 | 五行混 | 飞升之路，偶遇仙缘 |

> "灵根偏向"指**该地点更可能邂逅这些灵根的道侣**（70% 概率出对应灵根，30% 随机），但**品质仍按全局概率不变**（凡 50% / 仙 0.5%），不能定向刷红品。

#### 2.2.4 单次游历资源消耗

| 项目 | 消耗 |
|------|------|
| 灵石 | 当前境界基础灵石 ×1（金丹期 = 1万） |
| 时间 | **即时结算**（不消耗实时时间，符合放置游戏轻量化定位） |
| 仙玉 | 不消耗（基础游历不收仙玉） |

#### 2.2.5 游历产出（概率分布）

> **设计原则：游历是"道侣系统专属资源补给口"，不产出修为，不主产出灵石。** 玩家在 70% 非邂逅时段拿到的都是道侣养成相关的资源，强化"游历红尘 ≈ 为道侣储备"的内循环。灵石仅作为各类产出的少量附带，避免与普通刷图重复。

每次游历**必有产出**（避免"白跑"挫败），按以下概率随机一种结果：

| 概率 | 结果 | 主产出 | 附带 |
|------|------|--------|------|
| **30%** | 邂逅道侣 | 触发邂逅事件（4 选项剧情，品质按 2.3.1 全局概率） | — |
| **30%** | 道侣培养材料 | 见 2.2.5.1 子表 | 灵石 +500~1500 |
| **20%** | 红尘玉拾获 | 红尘玉 +20~80（按地点等级浮动） | 灵石 +200~800 |
| **10%** | 炼丹材料 | 五行灵草 ×1~3（品质按地点等级浮动） | 灵石 +300~1000 |
| **5%** | 修仙奇遇 | 文字小剧情 + 红尘玉 +50 / 情花种子 ×1 | 灵石 +500~2000 |
| **5%** | 修仙劫难 | 文字小剧情 + 小损失（灵石 -X / 短期 debuff） | — |

> **节奏验证**：30% 邂逅率 × 每天 5 次 = 期望 1.5 次邂逅/天；70% 资源产出 × 每天 5 次 = 期望 3.5 次为道侣养成"存料"。
> **"修仙劫难"5%** 是有意保留的负反馈，避免游历变成"100% 正反馈机制"，强化求道的不确定感。

##### 2.2.5.1 道侣培养材料子表（30% 主产出展开）

触发"道侣培养材料"产出时，进一步按以下概率分配：

| 子概率 | 物品 | 数量 | 用途 |
|--------|------|------|------|
| 35% | 凡品/灵品情花种子（相思藤、蝶恋花）随机 1 种 | ×1~2 | 礼物原料（3.3.3） |
| 25% | 玄品情花种子（月光兰）| ×1 | 中-上品礼物原料（高级地图限定） |
| 15% | 凡品~下品成品礼物（灵果蜜饯/彩珠串）随机 | ×1 | 直接赠送或留炼制 |
| 10% | 中品成品礼物（桃花酿/温玉香囊/童趣彩珠）随机 | ×1 | 按性格直接赠送 |
| 8% | 子女喂养灵草（凡-中品）| ×2~5 | 喂养（5.7.1） |
| 5% | **装备洗练材料**（附灵石 ×1 / 灵枢玉 ×1，各 50%）| ×1 | 复用项目现有装备附灵系统（`game/items.ts: awaken_stone / awaken_reroll`） |
| 1.8% | 地品/天品情花种子（并蒂莲、长情草）| ×1 | 极品礼物原料（高级地图首掉） |
| 0.2% | 红尘花种子 | ×1 | **仅七夕活动期间**触发，仙品礼物原料 |

> 装备洗练材料（附灵石/灵枢玉）是项目已有道具，把游历产出与玩家装备养成主线打通，让游历不仅服务于道侣系统，也支援装备线的"附灵刷词条"环节。

> **地点等级影响子分布**：高级地图（天雷荒原+/太虚仙境）的玄品+ 子概率翻倍，凡-灵品概率减半，让玩家有"刷高图换好种子"的策略性。**但邂逅道侣品质永远按全局概率不变**。

##### 2.2.5.2 红尘玉拾获子表（20% 主产出展开）

| 地点等级 | 红尘玉数量区间 |
|---------|--------------|
| 落霞古镇 (金丹) | +20 ~ +40 |
| 万妖山外围 (元婴) | +30 ~ +55 |
| 幽冥海岸 (化神) | +40 ~ +65 |
| 天雷荒原 (渡劫) | +50 ~ +75 |
| 太虚仙境 (大乘) | +60 ~ +80 |

##### 2.2.5.3 炼丹材料子表（10% 主产出展开）

掉落"五行灵草"（直接复用现有 `system-herb-pill-rework.md` 中的 7 种灵草）：

- 主灵草：按地图灵根偏向掉对应五行灵草（如落霞古镇 → 锐金草/厚土参 70%，其他 30%）
- 品质：地点等级越高，玄品+ 概率越高（与现有"野外掉落"规则一致）
- 数量：1~3 株
- **不掉情花系列**（情花专属于"道侣培养材料"分类）

> 这条把游历产出与现有炼丹/灵草系统打通，让游历也是炼"礼制"丹的原料补给（不仅是种植）。

#### 2.2.6 名册容量保护

| 情形 | 处理 |
|------|------|
| 名册已满 (5 位未结侣) | 游历可继续，但邂逅道侣概率 30% → **0%**，重新分配到其他 5 类产出（道侣培养材料 / 红尘玉 / 炼丹灵草 / 奇遇 / 劫难，按各自比例 30/20/10/5/5 重新归一） |
| 名册满 + 触发邂逅 | 由于上一行已归零，此情形理论不会发生；若因服务端竞态意外发生，则改为"擦肩而过"剧情提示 + 红尘玉 +30 补偿 |

#### 2.2.7 邂逅剧情库

设计 **20+ 段邂逅剧情**，每次触发从中随机一段。剧情结构：

```
【场景描述】
你来到【游历地点名】游历，忽闻幽幽古琴声从云雾深处传来......

【NPC 登场】
循声而去，只见一袭白衣女子立于古松之下，回眸一笑......
（自我介绍：道号 / 灵根 / 来历背景，全部根据本次随机品质生成）

【互动选择】
[A] 上前搭话    → +5 初始亲密度
[B] 远观致意    → +2 初始亲密度
[C] 拂袖离去    → 不录入花名册（跳过本次邂逅，但本次游历次数已消耗）
[D] 战斗试探    → 启动一场 PvE 表演战，胜利 +10 亲密度
```

#### 2.2.8 防滥用机制汇总

| 风险 | 对策 |
|------|------|

### 2.3 道侣品质与概率

#### 2.3.1 品质等级

| 品质 | 颜色 | 全局概率 | 子女资质上限 | 描述 |
|------|------|---------|-------------|------|
| 凡品 | 白 | 50% | 凡品 | 资质平庸的散修女子 |
| 下品 | 绿 | 25% | 下品 | 略有天资的修真者 |
| 中品 | 蓝 | 15% | 中品 | 中等灵根的宗门弟子 |
| 上品 | 紫 | 7% | 上品 | 大宗门嫡传女弟子 |
| 极品 | 金 | 2.5% | 极品 | 古老仙门继承者 |
| 仙品 | 红 | 0.5% | 仙品 | 上古传承的仙缘之人 |

> 期望：邂逅 200 次出 1 位仙品（红色）—— 玩家整个生涯期望获得 0-1 位红品道侣。

#### 2.3.2 品质决定的固定属性

每位道侣生成时按品质决定以下属性，**终身不变**：

| 属性 | 凡品 | 下品 | 中品 | 上品 | 极品 | 仙品 |
|------|------|------|------|------|------|------|
| 基础属性加成（同体阶段反哺本体） | +1% | +2% | +3% | +5% | +8% | +12% |
| 五行灵根纯度 | 五行混杂 | 单灵根 | 单灵根 | 单灵根 | 双灵根 30% | 五行混灵 5% |
| 修为加成上限（被动） | +3% | +5% | +8% | +12% | +18% | +25% |
| 子女资质上限 | 凡品 | 下品 | 中品 | 上品 | 极品 | 仙品 |
| 多胎率（双胎概率） | 0% | 1% | 3% | 5% | 10% | 20% |
| 三胎率 | 0% | 0% | 0% | 1% | 2% | 5% |
| 名册容量占用 | 1 | 1 | 1 | 1 | 1 | 1 |

#### 2.3.3 品质随机的属性（生成时随机）

每位道侣还会随机以下"个性"参数（影响互动玩法）：

```typescript
interface CompanionPersona {
  spiritualRoot: '金' | '木' | '水' | '火' | '土';   // 五行灵根
  personality: '冷艳' | '活泼' | '温柔' | '高傲' | '俏皮';  // 性格（决定喜好礼物）
  preferredGift: GiftType[];   // 喜爱的 3 种礼物（赠送有 +50% 加成）
  dislikedGift: GiftType[];    // 厌恶的 1 种礼物（赠送 -30% 亲密度，0% 概率小概率退还）
  appearance: AvatarId;        // 外观（决定 UI 头像）
}
```

性格 → 喜好礼物对照（示例）：

| 性格 | 喜爱礼物（+50% 亲密度） | 厌恶礼物（-30% 亲密度） |
|------|------------------------|------------------------|
| 冷艳 | 寒玉佩、月华丹 | 桃花酿 |
| 活泼 | 桃花酿、灵果蜜饯 | 寒玉佩 |
| 温柔 | 温玉香囊、灵果蜜饯 | 紫金钗 |
| 高傲 | 紫金钗、月华丹 | 童趣彩珠、彩珠串 |
| 俏皮 | 童趣彩珠、彩珠串 | 紫金钗 |

> 表中所有物品均在 **3.3.4 礼制丹方表**有完整定义（玩家可炼制获得），保证"性格匹配"系统在游戏内可达成闭环。
> **万能礼物**（并蒂莲心 / 鸳鸯玉佩 / 红尘仙缘簪）对**任何性格**都触发"喜爱"+50%。

### 3.1 亲密度获取来源

### 3.2 亲密度阶段

| 亲密度 | 阶段 | 解锁内容 |
|--------|------|---------|
| 0-100 | **萍水相逢** | 基础对话、赠礼界面 |
| 100 | **知己** | 解锁修为加成（被动 +3%~+25%，按品质） |
| 250 | **心动** | 解锁约会事件（每日 3 次随机） |

| **800** | **结发** | **解锁怀胎/生育系统** |
| 1200 | **心心相印** | 修为加成翻倍（按品质 +6% ~ +50%） |
| 4000 | **同体** | 道侣属性 30% 反哺本体 |
| 8000 | **道心** | 完美道侣终态，专属称号「比翼连理」 |

> **注**：100-250 阶段的解锁对未结侣对象同样有效；600+ 必须正式结侣才能继续推进。

### 3.3 礼物系统

#### 3.3.1 设计原则

**核心原则：道侣礼物完全独立于普通刷图掉落系统。**

- ❌ **普通地图战斗不掉落任何礼物或礼物原料**（避免玩家把"刷图"和"刷亲密度"绑定，破坏战斗节奏）
- ✅ **三条专属获取链**：
  1. **灵田种植 → 情花系列灵草**（生活技能耕作）
  2. **炼丹房合成 → "礼制"分类丹方**（生活技能炼制）
  3. **特殊活动地图 → 限时掉落成品礼物**（年度/节日限定）

> **设计意图：** 道侣养成走"生活技能 + 限时活动"双轨，盘活灵田/炼丹两个现有副系统，让玩家有"日常种植 → 周期合成 → 节庆收割"的稳定养成节奏，而不是被动等掉落。

#### 3.3.2 礼物分级总览

| 礼物等级 | 单次亲密度（基础） | 主要来源 | 示例 |
|---------|------------------|---------|------|
| 凡品 | +1 | 灵田直接产出（情花本体） | 相思藤、野花 |
| 下品 | +2 | 炼丹合成（凡-灵品灵草） | 灵果蜜饯、彩珠串 |
| 中品 | +3 | 炼丹合成（玄品灵草） | 桃花酿、温玉香囊、童趣彩珠 |
| 上品 | +5 | 炼丹合成（地品灵草）/ 节日地图掉落 | 寒玉佩、紫金钗、月华丹 |
| 极品 | +10 | 炼丹合成（天品灵草）/ 七夕地图掉落 | 并蒂莲心、鸳鸯玉佩 |
| 仙品 | +20 | **七夕年度限定活动地图** | 红尘仙缘簪（年限定） |

> **礼物不接入灵草品质系数**（2026-05-14 决策，简化设计）：礼物炼制时原料按"低品优先"自动消耗，成品不分品质。亲密度收益 = `baseIntimacy × 反应系数`（love ×1.5；仙品 love ×2.0；normal ×1.0；dislike 固定 -3）。例：用任意品质月光兰×3 + 玄水苔×5 炼制寒玉佩，赠送冷艳道侣 → 5 × 1.5 = **+8 亲密度**（取整）。
>
> **设计意图：** 灵草品质留给战斗丹/突破丹做养成深度，礼物侧只看"是否炼出 + 是否对路性格"两个维度，降低决策负担。

### 3.4 道侣加成（被动）

**前置：** 亲密度 ≥ 100（"知己"阶段）+ 已结侣
**机制：** **完全被动，无任何操作。** 已结侣后，玩家在历练打怪与挂机修炼时，修为获取自动增加 X%（取决于道侣品质）。无需按钮、无需启动、无时长限制。

> 备注：离线挂机系统当前处于测试阶段，本加成在离线场景下的具体生效方式待离线系统稳定后确认。MVP 阶段先在历练打怪场景生效。

| 道侣品质 | 修为加成（历练/挂机通用） |
|---------|------------------------|
| 凡品 | +3% |
| 下品 | +5% |
| 中品 | +8% |
| 上品 | +12% |
| 极品 | +18% |
| 仙品 | +25% |
| **心心相印（亲密度 1200）** | 上述 ×2 |

> 简化设计意图：万界仙途的核心循环是"挂机历练"，再加一个"道侣"主动按钮会让玩家觉得"不点亏了"，反而成为负担。改为常驻被动 buff 后，玩家无感享受加成，符合放置修仙的轻量化定位。
> 1200 亲密度（心心相印）后加成翻倍，是亲密度长线培养的最大数值激励。

#### 3.4.1 陪伴亲密度（每日自动累积）

为补偿删除"双修每日 +20"后的亲密度地板，**已正式结侣后每日自动获得 +20 陪伴亲密度**，玩家上线时自动结算前一日累积，最多累计 7 天 = 140。

| 项目 | 数值 |
|------|------|
| 每日陪伴亲密度 | +20（自动） |
| 离线累计上限 | 7 天 = 140 |
| 触发方式 | 玩家上线时自动结算（与现有 `daily-reset` cron 同步） |
| 前置条件 | **已正式结侣**（仅对正式道侣 `is_official = true` 生效） |

> 这条相当于"夫妻日常陪伴自然增进"，作为结侣后的核心福利之一，强化"结侣即开启稳定增长"的体验。
> **未结侣阶段（100-600）只能靠礼物/约会/子女积累亲密度**，避免玩家同时养 5 位红颜每人 +20/天，导致一周内全部红颜达到结侣门槛、破坏阶段节奏。

### 3.5 约会事件

**前置：** 亲密度 ≥ 250（"心动"阶段）
**机制：** 每日 3 次随机约会事件，类似随机事件系统但走"道侣专属事件池"。

事件示例：

```
【清风踏月】
你与【慕容雪】共游清风月下，她忽然问：
"若你为剑修，可知何为剑心？"
[A] 剑心即道心      → 答对 +15 亲密度，获得 100 灵石
[B] 剑心即杀伐      → 答错 +3 亲密度
[C] 剑心即杀身成仁  → 答对 +15 亲密度，获得 月华丹 ×1（用于赠送道侣）
```

事件类型分布（每日 3 次随机）：

## 四、和离机制

### 4.1 设计原则

- **允许和离**：避免玩家因"早期选错"被永久绑定，留出试错空间
- **代价高昂**：防止玩家随意切换道侣"刷资质"
- **冷却保护**：和离后必须等冷却才能再次结侣，避免脚踩两条船

### 4.2 和离代价

| 项目 | 代价 |
|------|------|

| 仙缘印记 | 重置为 LV 0（无效） |
| 子女归属 | 子女**全部由玩家保留**（NPC 道侣离开） |
| 冷却 | 和离后 **1 个自然日**内不能正式结侣（可继续邂逅+培养亲密度，但不能结侣） |
| 历史记录 | 永久记录在角色档案中（"曾有红尘伴侣 N 位"） |

## 五、子女系统（核心模块）

### 5.1 设计原则

- **稀缺**：单次怀胎 24 小时，总子女上限 5 名（在家 + 已离家 合计）
- **传承感**：资质继承父母，强化"血脉"认同
- **可玩性**：子女可装备、可培养、可出战，是道侣系统的"长期载体"
- **自由度**：子女成年后**由玩家决定是否出战**（不强制离家），支持长期"传家宝"和短期"代代换"两种玩法

### 5.2 怀胎系统

#### 5.2.1 触发条件

```
亲密度 ≥ 800（结发阶段）
+ 已正式结侣
+ 当前未怀胎
+ 子女总数（含养育中） < 3
```

#### 5.2.2 怀胎流程

```
道侣 → [求子] 按钮
    ↓
消耗：金莲花露 ×1 + 灵石 ×100万
    ↓
怀胎期：24 小时（实时）
    ↓
怀胎期间：
  - 道侣面板显示"怀胎中"
  - 修为加成与陪伴亲密度暂停（道侣专心养胎）
  - 仙缘印记 buff 仍生效
  - 可购买"安胎符"加速 ×2（仙玉道具）
    ↓
24 小时后自动出生 → 弹窗显示子女信息
```

**金莲花露获取来源（设计内定义）：**
- 红尘玉商店：3000 红尘玉 ×1（每周限购 2 个）
- 节日活动地图掉落（七夕鹊桥幻境噩梦难度首通保底 ×1）

> 金莲花露作为生育门槛物品，定价相对高昂以维持生子的"仪式感"，但同时提供 4 种渠道避免完全卡死玩家。

### 5.3 子女资质生成

#### 5.3.1 资质等级

```
凡品 → 下品 → 中品 → 上品 → 极品 → 仙品 → 圣品（最高）
```

> 注意：子女资质上限 = 父母（道侣）品质上限。**圣品仅通过"夺天造化丹"重铸触发**（极稀有）。

#### 5.3.2 资质生成公式

```typescript
function generateChildAptitude(player: Character, companion: Companion): ChildAptitude {
  const ceiling = companion.qualityIndex; // 0=凡品 ... 5=仙品
  const floor = Math.floor(player.aptitudeIndex * 0.5); // 玩家自身资质 50% 作下限
  
  // 在 [floor, ceiling] 范围内随机
  const baseAptitude = randomBetween(floor, ceiling);
  
  // 5% 概率突破上限 +1（"血脉觉醒"）
  if (random() < 0.05 && baseAptitude < 6) {
    return { aptitude: baseAptitude + 1, awakened: true };
  }
  
  return { aptitude: baseAptitude, awakened: false };
}
```

**示例：** 玩家本体资质 上品（4），道侣极品（金，4）
- 下限 = floor(4 × 0.5) = 2 = 中品
- 上限 = 4 = 极品
- 子女资质区间：中品 ~ 极品
- 5% 概率出现"血脉觉醒"突破到仙品

#### 5.3.3 五行灵根继承

```typescript
function generateChildElement(player: Character, companion: Companion): Element {
  const r = random();
  
  if (r < 0.45) return player.element;        // 45% 继承玩家
  if (r < 0.90) return companion.element;     // 45% 继承道侣
  if (r < 0.99) return randomElement();       // 9% 突变随机
  
  // 1% 双灵根 / 五行混灵
  return rollMutationElement(); 
}
```

#### 5.3.4 性别与外观

- 性别：50% 男 / 50% 女
- 道号：自动生成（"〇〇儿"格式，可改名）
- 外观：道侣外观池中随机一个（同性别）

### 5.4 子女成长阶段

| 等级 | 阶段 | 持续时间 | 玩法 |
|------|------|---------|------|
| 1-10 | 婴幼期 | cron 每日 +1 级 | 喂养（消耗灵草，每日上限 5 次），不可出战 |

### 5.5 子女属性

#### 5.5.1 基础属性公式（v2.0 真双人战斗）

> **架构升级**：v1.x 的「子女属性聚合到本体」已废弃。当前子女是 **`duoBattleEngine` 的独立战斗单位**（独立 hp / buff / 功法 CD，能被怪物攻击和击杀）。

**子女出生时面板**（`server/utils/child.ts` `calcChildBaseStats`）：
```typescript
maxHp = floor(200 + level × 250 × 资质倍率)
atk   = floor(20  + level × 22  × 资质倍率)
def   = floor(15  + level × 10  × 资质倍率)
spd   = floor(30  + level × 2.5 × 资质倍率)
// 会心率/会心伤害/闪避：出生时按资质浮动滚一次，升级不重滚
// 神识/控抗：随等级 × 资质倍率成长
```

**入战缩水**（`server/api/battle/fight.post.ts` `_assistPrep` → `_duo_assist_stats`）：
```
子女最终入战属性 = (子女面板 + 装备主属性 + 装备副词条) × (1 + 天赋百分比) × 阶段倍率
```
- **阶段倍率**：少年 0.5 / 青年 0.8 / 成年 1.0（2026-05-13 调整，原 0.3 / 0.6 / 1.0）
- **无对标本体 cap**（2026-05-13 起去除）：高资质子女成年后可超越本体面板，作为「次主力」独立扛伤输出

**与 v1.x 差异**：
- ❌ 没有「父母属性继承 10%」（v1.x 设计未落地）
- ❌ 不是「加给玩家本体」，而是子女**自己**以缩水后的属性独立战斗
- ❌ 不再对标本体 70% cap（2026-05-13 起移除）
- ✅ 装备加成 + 天赋百分比 仍生效，并随阶段倍率一起入战

#### 5.6.3 子女天赋觉醒

##### 觉醒规则

- 每 20 等级觉醒 1 个被动天赋，最多 **7 个**槽位（对应 20/40/60/80/100/120/140 级）
- 觉醒**完全随机**，玩家可消耗"洗髓丹"重铸单个槽位
- 高资质子女天赋池权重向高稀有度倾斜
- **每个天赋本质是一个 `PassiveEffect` 实例**，与玩家功法被动同源（参考 `server/engine/skillData.ts: PassiveEffect`），战斗引擎零改造直接消费

##### 设计约束（与现有战斗引擎对齐）

| 约束 | 说明 |
|------|------|
| 持续效果单位 | 一律用 **回合（turn）**，无"秒"概念（万界仙途是回合制） |
| 字段范围 | 仅使用 `PassiveEffect` 已定义字段（atk%/def%/hp%/spd%/crit_rate/crit_dmg/dodge/lifesteal/resists/regen/reflect/dot 触发/复活等） |
| 控制效果 | 仅复用现有 debuff 类型（burn/poison/bleed/freeze/stun/silence/root/brittle/atk_down/slow） |
| 数值上限 | 单个天赋数值显著弱于玩家紫/金/红被动功法（避免子女反超本体） |
| 不引入 | 不引入"召唤""阶段变身""真实秒数""完全免疫"等引擎暂不支持机制 |

##### 天赋稀有度

| 稀有度 | 配色 | 整体数值 | 特点 |
|--------|------|---------|------|
| 普通 (Common) | 白 | 单字段低数值 | 入门级 |

##### 重铸（洗髓丹）

```
子女详情 → 天赋槽位 → [洗髓]
    ↓
消耗"洗髓丹" ×1（无论后续是否替换都已消耗）
来源：仙玉商城 ¥30 / 节日活动兑换 / 红尘玉商店 1500
    ↓
按当前资质权重重新随机一个天赋
    ↓
弹窗预览新旧对比
    ├── [接受新天赋]  → 替换旧天赋
    └── [保留旧天赋]  → 不替换，但洗髓丹仍消耗
```

> **保留旧天赋 ≠ 退还洗髓丹。** 这是给玩家"万一新天赋更差"的兜底（避免被随机性强制接受劣势天赋），但消耗已付出，**保证"刷传说"不能零成本进行**。
> 仅本次开洗的当前对比有效，确认后无法回滚。

| 风险 | 应对 |

#### 5.7.2 子女血脉功法（V2 改版，2026-05-14）

子女**不学习玩家功法**，而是出生时根据灵根自动觉醒**独有功法**，并随等级解锁更多槽位。功法分六大类型，每类对应一种战斗职能：

> **V2 改版要点（与 V1.8 差异）**
> - 类型从 4 类（攻击/肉盾/回复/Buff）扩展为 **6 类**（新增 **嘲讽** + **减益**）
> - 数量从 20 个扩展为 **31 个**（6 类 × 5 品质 = 30，攻击型红品再加 1 个连击型 = 31）
> - 槽位从单一固定 1 个改为 **多槽位**（出生 1 槽 + 等级解锁 2-3 槽，圣品独享第 4 槽）
> - 新增 **「血脉重铸丹」** 可单独重抽指定槽位（不再必须用资质丹）

##### 槽位与解锁

| 槽位 | 解锁等级 | 资质要求 | 说明 |
|-----|---------|---------|------|
| 槽 1 | Lv.1 | 所有资质 | 出生时按灵根+资质自动 pick |
| 槽 2 | Lv.100 | 所有资质 | 喂养至 Lv.100 时升级 API 自动解锁 |
| 槽 3 | Lv.200 | 所有资质 | 喂养至 Lv.200 时升级 API 自动解锁 |
| 槽 4 | Lv.300 | **圣品 ★★★★★★（aptitude=6）独享** | 圣品养成深度差异化奖励 |

> 多槽位之间**类型自由搭配**（如槽 1=攻击 + 槽 2=回复 + 槽 3=嘲讽 完全合法），玩家通过血脉重铸丹按需调整。

##### 功法升级

子女功法跟随子女等级**自然升级**（不消耗资源），等级上限由资质决定（参考 5.5.2 资质对应的等级上限）。功法效果数值随等级线性提升：

```
功法实际效果 = 基础数值 × (1 + (子女等级 / 100) × 0.5)
```

例：子女 50 级时，绿品「童子拳」实际倍率 = 1.10 × (1 + 0.25) = **1.375**

##### 数据库字段对应

`children.learned_skills` JSONB 字段格式（V2 改版加 `slot` 字段）：

```json
[
  { "skill_id": "ch_atk_purple", "level": 1, "slot": 1, "type": "innate" },
  { "skill_id": "ch_heal_purple", "level": 1, "slot": 2, "type": "innate" },
  { "skill_id": "ch_taunt_purple", "level": 1, "slot": 3, "type": "innate" }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `skill_id` | string | 对应 `CHILD_SKILL_MAP[id]` 查 type/effect 等 |
| `level` | number | 当前默认 1（数值缩放走 `calcSkillLevelMultiplier(子女等级)`） |
| `slot` | number | 1/2/3/4，槽位编号（V2 新增） |
| `type` | string | 历史字段，固定 `'innate'`（实际 type 取 `CHILD_SKILL_MAP[id].type`） |

> 老数据（V1）无 `slot` 字段时，detail/feedChild/reroll-skill 等 API 按数组顺序自动补 `slot = index + 1`。

##### 战斗端处理

子女功法在 `fight.post.ts` 中按 type 分流到不同执行路径：

| Skill type | 处理路径 | 说明 |
|-----------|---------|------|
| `passive` | `_assistPrep` 静态加成 | 与天赋走同一 PassiveEffect 管线，聚合到 atkPct/defPct/critRate/resist 等 |
| `divine` | CD 池（取第 1 个 divine 进 `assistSkillCd`） | 在 duoBattleEngine.assistAction 按 cdTurns 轮转 cast |
| `active` | 替代普攻（multiplier > 1） | 子女普攻倍率提升 |

> 当前实现：若一个子女学到多个 divine 功法，只有**第 1 个** divine 进 CD 池（如三槽都是 divine，槽 2/3 仅 description 展示不生效）。多 divine 轮转留 Phase 4。

##### 与玩家功法的关系

| 维度 | 玩家功法 | 子女血脉功法（V2） |
|------|---------|------------------|
| 学习 | 主动选择 + 消耗材料 | 出生时按灵根+资质自动觉醒，等级解锁更多槽位 |
| 数量 | 多个槽位（按品质） | 3 个槽（圣品 4 个） |
| 升级 | 消耗功法残页 | 跟随等级自然提升 |
| 重铸 | 不可 | **可（消耗血脉重铸丹按槽位重抽）** |
| 触发 | 玩家释放 | 子女助战时自动释放（按 CD） |
| 数据 | `character_skills` | `children.learned_skills` |

##### 重铸：血脉重铸丹

新增道具 `bloodline_reroll_pill`「血脉重铸丹」：

| 字段 | 值 |
|------|---|
| 道具 ID | `bloodline_reroll_pill` |
| 名称 | 血脉重铸丹 |
| 描述 | 重抽子女指定槽位的血脉功法（按当前灵根+资质重选） |
| 获取 | 红尘玉商店：800 红尘玉/颗，每周限 20 颗 |
| API | `POST /api/child/reroll-skill { child_id, slot, accept_new? }` |

> 与「夺天造化丹」（资质重铸，3000 红尘玉/颗）的区别：资质丹会**重生所有槽位**，重铸丹只动**指定槽位**。日常优化首选重铸丹，资质突破才用资质丹。

> 设计要点：子女功法是"血脉天授"的辨识度，与天赋池（被动 buff）形成"主动技 + 被动 buff + 嘲讽/减益辅助"的三层成长。多槽位 + 重铸丹让玩家可以按 build 主动雕琢，而不是完全靠出生抽卡。

#### 5.7.3 资质重铸

```
子女详情 → [洗练]
    ↓
消耗"夺天造化丹" ×1（极稀有，需活动/秘境获取）
    ↓
重新随机资质（仍受父母品质上限约束）
```

### 5.8 成年后选择

子女达到 100 级（成年）后弹窗：

```
【慕容雪儿】已成年！
她道："父亲（母亲），孩儿欲......"
[ A ] 留在家中助战 → 继续作为助战单位
[ B ] 外出历练    → 脱离助战，每 3 天自动回家给予 1 次永久属性 +0.5% 加成
```

> **选择 B 的长期回报**：每 3 天回家一次永久 +0.5%（最多累计 +20% = 40 次 ≈ 120 天），鼓励长期挂机玩家。
> **可中断**：在子女详情卡片可随时「召回回家」（`POST /api/child/recall-home`），已累计 buff 完整保留；召回后可重新设为助战，或再次选 B 继续累积。

### 5.9 子女出战与现有秘境联动

- 子女作为助战单位**自动加入现有秘境组队**（占用玩家本体的位置 — 玩家+子女算 1 个队伍位）
- 子女出战的秘境局获得**双倍亲密度积累**到正式道侣（"孩子在父母面前表现"）
- 子女单独获得 +20 红尘玉/秘境（鼓励带娃出战）

### 5.10 放弃子女（2026-05-13 新增）

子女名册总上限 5（在家 + 离家合计）。当离家子女把 +20% 上限攒满后，玩家可主动**放弃子女**释放槽位，再育新子女继续累积全局 buff（理论上限可循环突破 80%）。

#### 操作流程

```
子女详情 → [危险操作 · 放弃子女]
    ↓
消耗 灵石（按境界缩放）
    ↓
DELETE FROM children（含装备 ON DELETE CASCADE）
写 child_abandon_history 历史表（仅做记录）
若该子女正在助战 → 清掉 characters.battling_child_id
```

#### 代价表

| 境界 | tier | 灵石代价 |
| ---- | ---- | -------- |
| 金丹 | 3 | 25 万 |
| 元婴 | 4 | 50 万 |
| 化神 | 5 | 75 万 |
| 炼虚 | 6 | 100 万 |

> 计算式：`5000 × 50 × max(1, realm_tier - 2)`，约为和离代价的一半。

#### 规则要点

- **buff 立即清零**：离家子女累计的 `permanent_buff_pct` 一并消失，**不转存到本体**。否则数值上限失控。
- **不可撤销**：DELETE 后无法恢复，UI 强制二次确认弹窗。
- **保留历史**：写入 `child_abandon_history`，含名字 / 资质 / 等级 / 阶段 / 被弃前 buff 快照，可做"曾放弃 N 位子女"成就追踪。
- **释放槽位**：子女总数 -1，可继续怀胎生育。
- **道侣关系不变**：被弃子女的 `parent_companion_id` 仍在 history 里追溯，但不再占用现存道侣的子女计数。

#### 设计意图

- 提供"刷子女"的明确路径，让长期玩家在攒满 4 × 20% = 80% 后仍有数值成长空间
- 通过灵石代价 + 怀胎重新养成时间双重门槛控制频率，避免一天刷 10 个新生儿
- "缘尽于此" 主题与道侣"和离"机制一致，保持世界观

```typescript
// Response
{

  outcome: {
    type: 'encounter' | 'gift_material' | 'red_jade' | 'herb' | 'fortune' | 'mishap',
    
    // type='encounter' 时
    encounter?: {
      event_id: number,                 // pending 事件 ID
      script: { id, title, scene },
      persona: { quality, root, personality, avatar },
    },
    // type='gift_material' 时（道侣培养材料）
    gift_materials?: [{ type: 'seed' | 'gift' | 'herb' | 'jade_token', id, quantity }],
    // type='red_jade' 时
    red_jade_gained?: number,
    // type='herb' 时（炼丹原料）
    herbs?: [{ herb_id, quality, quantity }],
    // type='fortune' / 'mishap' 时
    text?: string,
    rewards_or_penalties?: { red_jade?: number, seeds?: [...], stones?: number, debuffs?: [...] },
    
    // 所有产出共享
    spirit_stone_bonus?: number,        // 附带灵石（劫难时为负数）
  },
  daily_remaining: number,           // 剩余次数
}

// 服务端流程
async function startExpedition(characterId: number, locationId: string) {
  const char = await getCharacter(characterId);
  
  // 前置校验
  assert(char.realm_tier >= 3, 'REALM_TOO_LOW');
  assert(char.expedition_count_today < dailyMax(char), 'DAILY_LIMIT');
  assert(canAccessLocation(char.realm_tier, locationId), 'LOCATION_LOCKED');
  assert(char.spirit_stone >= expeditionCost(char.realm_tier), 'INSUFFICIENT_STONE');
  
  // 扣灵石、扣次数
  await deductCost(characterId, expeditionCost(char.realm_tier));
  await incrementExpeditionCount(characterId);
  
  // 滚产出类型
  let outcomeType = rollOutcome();   // 30/25/20/10/5/10
  
  // 名册满时邂逅率归零
  if (outcomeType === 'encounter' && await getCompanionCount(characterId, false) >= 5) {
    outcomeType = rerollWithoutEncounter();   // 重滚（剔除 encounter）
  }
  
  // 按 outcomeType 分发
  return dispatchOutcome(outcomeType, characterId, locationId);
}
```

## 九、数值平衡

### 9.1 时间投入预估

以**完整道侣养成路径**为例（不同品质道侣到达"心心相印"阶段所需天数）：

| 道侣品质 | 邂逅期望 | 结侣前 (~80/天) | 结侣后 (~130/天) | 到达 600 (结侣) | 到达 800 (生子) | 到达 1200 (修为加成翻倍) |
|---------|---------|----------------|------------------|----------------|------------------|---------------------|
| 凡品 | 容易 | 礼物+约会 | + 陪伴+子女 | 8 天 | 10 天 | 13 天 |
| 下品 | 容易 | 礼物+约会 | + 陪伴+子女 | 8 天 | 10 天 | 13 天 |
| 中品 | 中等 | 礼物+约会 | + 陪伴+子女 | 8 天 | 10 天 | 13 天 |
| 上品 | 困难 | 礼物+约会 | + 陪伴+子女 | 8 天 | 10 天 | 13 天 |
| 极品 | 极难 | 礼物+约会 | + 陪伴+子女 | 8 天 | 10 天 | 13 天 |
| 仙品 | 几乎不可 | 礼物+约会 | + 陪伴+子女 | 8 天 | 10 天 | 13 天 |

> 不论品质，到达各阶段时长相同。**唯一差异是邂逅期望次数**：仙品玩家可能整个生涯只能遇到 0-1 位。
> 道侣加成被动化后，结侣是关键节点 — 一旦正式结侣，"陪伴亲密度+20/天"自动生效，养成效率从 80/天 → 130/天 提速 60%。

### 9.2 战力提升预估

| 阶段 | 来源 | 战力提升 |
|------|------|---------|
| 结侣 | 仙缘印记 LV 1 | +3% 全属性 |
| 仙缘印记满级 | LV 5 | +15% 全属性 |
| 同体（亲密度 4000） | 道侣属性 30% 反哺本体 | +10~20% 全属性 |
| 子女出战 (上品资质 成年) | 真双人战斗，子女作为独立单位 100% 面板入场（无对标本体 cap） | 视资质浮动；上品成年子女面板可接近本体，仙圣品可超越 |
| 离家子女回家加成 (40 次) | 永久 +0.5% × 40 | +20% 全属性 |

**理论上限叠加：** 一名玩家配齐"仙品道侣 + 极品子女出战 + 长期离家子女回家"后，全属性可达 +60%~90%，但**需要 200+ 天的长线养成**。子女作为真双人战斗的次主力，能显著影响战局；自然平衡靠资质稀有度（凡品 1.0× ~ 圣品 5.0× 的属性倍率差）和"同时仅 1 名出战"实现。

### 9.3 防破坏机制

| 风险 | 应对 |
|------|------|

## 十二、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 单角色游戏引入伴生角色破坏定位 | "我是修仙散修，不是养老婆的" | 道侣定位为"加成系统 + 长线情感"，**不直接进入战斗**，仅通过道侣加成/仙缘印记/属性反哺等被动方式提供加成 |
| 子女属性失衡 | 后期子女超越本体 | 设计立场已转变（2026-05-13）：允许高资质子女超越本体，定位为"真双人战斗的次主力"。靠资质稀有度（凡 1.0 ~ 圣 5.0 倍率）和"同时仅 1 名出战"形成自然平衡 |
| 离婚滥用 | 玩家不停换道侣刷资质 | 高昂代价（红尘解 ¥30 + 50w 灵石）+ 1 天冷却 + 公开广播 |
| 仙品道侣过度稀缺 | 99% 玩家拿不到 | **仙品作为"梦想"而非"必备"** — 上品（紫）道侣已能享受全部玩法，仙品仅在多胎率/子女资质上限上有差异；邂逅 200 次（约 130 天游历）期望出 1 位仙品，节奏合理 |
| 子女系统让萌新困惑 | 复杂度爆炸 | 严格按解锁阶段渐进式开放，金丹期才解锁邂逅 |
| 和离公开广播过激 | 玩家社交压力 | 提供"低调和离"选项（消耗 +1 个红尘解免广播） |
| 怀胎 24 小时太长 | 节奏感不佳 | 提供安胎符（仙玉道具，¥6 加速 ×2）+ 洞府"暖玉床"减时间 |
| 无法决定道侣性别 | 部分玩家诉求 | 后期可拓展"男道侣"（双轨剧情库），MVP 阶段统一为女性 |
| 多胎/三胎导致子女井喷 | 养育槽位 3 个被快速占满 | 多胎概率严格控制；可选择"舍弃"未育子女（无代价） |
| 资质重铸玩家无法接受随机 | 体验挫败 | 提供"随机但保底" — 重铸结果若低于当前则保留旧资质 |

### 附录 C：开发问答（FAQ）

**Q1: 为什么道侣不是另一个 PvP 玩家？**
A: 万界仙途是单角色挂机修仙，跨玩家的 1v1 强绑定会引入大量同步问题（在线状态、跨账号转移子女等）。道侣作为 NPC 更符合"游戏内陪伴"的定位，剧情和情感设计也更可控。

**Q2: 玩家可以同时与多个未结侣对象互动是否过于"花心"？**
A: 修仙小说中"红颜知己"是常见桥段（参考《诛仙》《凡人修仙传》等）。同时只能正式结侣 1 位，其他未结侣对象**仅能赠礼+培养亲密度，不享有"陪伴亲密度+20/天"和生子福利**，符合修仙世界观。

**Q3: 子女出战是否会让 PvP（论道台）失衡？**
A: 2026-05-13 起去除 70% cap，子女可作为真双人战斗的次主力，面板可与本体并肩。但 PvP 平衡靠资质稀有度（凡品 1.0 ~ 圣品 5.0 倍率）+ 双方都只能带 1 名子女实现：氪不出仙圣品子女就只能用稀有度匹配的凡-上品子女，整体仍均衡。

**Q4: 仙品道侣过于稀有，普通玩家是否会感到挫败？**
A: 设计上"仙品 = 终极目标"，类似 SSR 卡。普通玩家拿到上品道侣已经能享受全部玩法，仙品仅在修为加成、子女资质上限、多胎率上有差异，不阻断进度。

**Q5: 子女成年后离家是否过于"残忍"？**
A: 离家是**玩家可选**项（推荐选 A 留家）。选择 B 离家是"长期挂机奖励"路径，每月 +0.5% 永久属性，类似"传承"概念，对长期玩家更友好。

**Q6: 怀胎 24 小时是否合理？**
A: 借鉴《一念逍遥》生子周期（24-72 小时不等）。24 小时是兼顾仪式感与玩家节奏的下限值（2026-05-13 由原 48h 调整）。可通过仙玉道具加速，或洞府"暖玉床"建筑减时间。
