# 技能石系统设计（Skill Stone / 功法镶嵌）

> 最后更新: 2026-04-24
> 版本: v0.2 — 完全替代版（废弃旧功法）
> 设计目标: **彻底废弃现有 46 个固定功法，改为「通用功法书 + 可镶嵌技能石」的纯组合体系。游戏中不再存在"烈焰剑诀"这种固定名字的功法，所有效果都由玩家镶嵌的石头决定。**

---

## 一、设计哲学

### 1.1 核心痛点（为什么要全盘替换）

现有功法体系的根本问题不是"功法太少"，而是**"功法本身就是静态数据"**：
- 46 个功法 = 46 条写死的 JSON 配置，每条几乎不可调整
- 玩家感受到的选择只是"从池子里抽到哪个"，不是"我想要什么"
- 流派固化：火系就是灼烧、水系就是冻结，**没有 Build 的空间**
- 扩展成本高：每新增一种玩法就得手写一条功法配置 + 测试 + 平衡

**新系统的断代式取舍**：
- 旧的 `skill_templates.active_skills / divine_powers / passives` 三张表**全部废弃**
- 现有 46 个功法作为"历史快照"保留在设计文档里，**但运行时不再存在**
- 玩家库里的旧功法统一执行**迁移脚本**（见第十章）转成"书 + 石头"
- 未来所有"功法"的新增 = **新增一颗石头**，而不是新增一个功法

### 1.2 设计原则

1. **纯石头驱动**：功法书是"空壳容器"，**不镶嵌核心石等同于不存在**。书本身不带任何效果
2. **书只提供三件事**：① 类型（主修/神通/被动）② 品质（决定孔位数）③ 元素倾向（装对应元素石头获得 +20% bonus）
3. **上限可控**：每本书最多 5 孔；孔位类型固定，防止堆叠同类
4. **组合爆炸但非无限**：强制互斥关系（群攻 ↔ 多段互斥）、核心石/触发石/质变石各自唯一
5. **伪 PoE 不是 PoE**：回合制 + 自动战斗，不需要放置树那种深度，**5 孔 = 目标复杂度上限**
6. **迁移平滑**：老玩家登录时自动执行一次等价迁移，获得相当价值的书 + 石头包，无损失

### 1.3 🔒 数值护栏（与 v2.0 上限对齐）

| 维度 | 单石上限 | 5 孔叠加上限 | 与引擎 cap 的关系 |
|------|---------|------------|----------------|
| 基础倍率加成 | +80% | +150%（clamp） | 不突破仙品功法天花板 10x |
| Debuff 概率加成 | +25% | +40%（clamp） | 配合 debuff_vs_resist 计算 |
| 持续回合加成 | +2 回合 | +3 回合（clamp） | DOT 封顶 6 回合 |
| 目标数加成 | 单→多(2~3) | 多→群攻 | 群攻后不可再加目标石 |
| 溢出/连锁 | 1 次 | 1 次（不可叠加） | 防止无限链式 |
| 吸血/回血转化 | 20% | 30%（clamp） | 与引擎 lifesteal cap 对齐 |

**核心约束**：技能石是**形态变换**，不是数值膨胀。5 孔满镶嵌的红品书 ≤ 1.3x 同品质传统功法；强度来自"能匹配 Build 的形态"而非"数值碾压"。

## 二、功法书结构

### 2.1 功法书分类

书只有 **3 种类型 × 6 种品质 × 6 种元素倾向 = 仅 108 个 SKU**（替代旧 46 个独立功法）：

| 类型 | 作用 | 装备槽位 |
|------|------|---------|
| **主修书**（Active Book） | 替代普攻，每回合自动施展 | 1 槽 |
| **神通书**（Divine Book） | 有 CD 的爆发/控制/回复/buff | 3 槽 |
| **被动书**（Passive Book） | 永久加成 | 3 槽 |

**元素倾向**：金/木/水/火/土/无。元素倾向不决定效果，只决定"装对应元素的石头时获得 +20% bonus"，与现有灵根匹配规则保持兼容。

**示例**：「玄品·火系神通残卷」= 玄品书 + 火元素倾向 + 神通类型。装火系核心石时触发 bonus；装水系核心石也能用，但无 bonus。

### 2.2 孔位布局

| 品质 | 开放孔位 | 孔位类型 | 备注 |
|------|---------|---------|------|
| 凡品（白） | 1 | 1 核心 | 只能装白/绿品核心石，入门用 |
| 灵品（绿） | 2 | 1 核心 + 1 增幅 | |
| 玄品（蓝） | 3 | 1 核心 + 2 增幅 | |
| 地品（紫） | 4 | 1 核心 + 2 增幅 + 1 触发 | |
| 天品（金） | 5 | 1 核心 + 2 增幅 + 1 触发 + 1 质变 | |
| 仙品（红） | 5 | 1 核心 + 2 增幅 + 1 触发 + 1 质变（孔位可装更高品石头） | |

### 2.3 孔位类型定义

| 类型 | 颜色 | 数量 | 作用 |
|------|------|------|------|
| **核心孔**（Core） | 红 | 每书恰好 1 | 决定技能本质：伤害 / 控制 / 回复 / buff / 反伤 |
| **增幅孔**（Amp） | 蓝 | 0~2 | 改数值：倍率+ / 概率+ / 持续+ / 目标+ |
| **触发孔**（Trigger） | 黄 | 0~1 | 改触发条件：CD- / 血量触发 / 暴击触发 |
| **质变孔**（Ultimate） | 紫 | 0~1 | 改形态：溢出转化 / 吸血 / 必中 / 连锁 |

### 2.4 关键规则

- **核心石必装**：未装核心石的书**完全无效**，战斗中跳过不触发
- **书不带任何基础倍率**：所有倍率/效果均来自核心石 + 增幅石
- **增幅孔可同类**：2 个倍率石可叠加（但受 clamp，详见 1.3 护栏）
- **触发/质变唯一**：同一本书不能有 2 个"暴击时触发"或 2 个"溢出"，防止暴力链
- **镶嵌可拆卸**：消耗"拆石丹"（白品 10 灵石，每升一品 ×5）；**不消耗石头**，鼓励玩家试错

## 四、新手默认配方（Starter Builds）

旧系统没了，为避免新玩家"看着一本空书不知道装什么"，内置 **5 套新手默认配方**。玩家首次获得对应品质的书时，可**一键套用**推荐配方（也可立刻拆掉自己玩）。

### 4.1 新手·单体灼烧（绿品主修书）
- 核心：`core_burn`（灼烧，100% 伤害 + 附灼烧）
- 增幅 1：`amp_dmg_small`（+30% → 130%）
- 最终效果：130% 伤害 + 灼烧 debuff（火元素书 → +20% bonus → 156%）

### 4.2 入门·双目标灼烧（紫品神通书）
- 核心：`core_flame_slash`（烈焰 200%）
- 增幅 1：`amp_dmg_mid`（+50% → 300%）
- 增幅 2：`amp_target_2`（双目标）
- 触发：（空）
- 最终效果：[2 目标] 300% 伤害

### 4.3 进阶·多段中毒（紫品神通书）
- 核心：`core_poison`（中毒 100%）
- 增幅 1：`amp_multi_hit_3`（三段）
- 增幅 2：`amp_chance`（+15% 概率）
- 触发：（空）
- 最终效果：3 段 × 100%，每段独立判定中毒

### 4.4 高阶·溢出斩杀（金品神通书）
- 核心：`core_bleed`（流血 100%）
- 增幅 1：`amp_multi_hit_5`（五段）
- 增幅 2：`amp_chance_mega`（+25%）
- 触发：（空）
- 质变：`ult_overflow_dmg`（溢出斩杀）
- 最终效果：5 段连击，目标死后溢出到下一只

### 4.5 仙级·群体控场（红品神通书）
- 核心：`core_timestop`（必定冻结 2 回合）
- 增幅 1：`amp_target_all`（群攻）
- 增幅 2：`amp_duration`（+1 回合 → 冻结 3 回合）
- 触发：`trigger_first_hit`（前 3 回合伤害 +40%）
- 质变：`ult_chain`（连锁 60% 伤害到第二目标）
- 最终效果：群体冻结 3 回合 + 先发制人 + 连锁伤害

---

**注**：这些"默认配方"只是入门推荐，不是固定死的。玩家可以随时拆石头组合自己的 Build——比如把 4.5 的核心换成 `core_chaos_sword`，立刻从控场变成爆发 AOE。

## 五、扩展 BD 示范（旧系统做不出的玩法）

### 5.1 BD-A「治疗 = 攻击」流（用户原始灵感）

**功法书**：紫品主修「生生不息」
- 核心：`core_heal_instant`（回复 200% ATK）
- 增幅 1：`amp_target_2`（回复 2 人，用户例子）
- 增幅 2：`amp_dmg_large`（+80% → 回复 280% ATK）
- 触发：`trigger_on_debuff`（敌人每条 debuff，回复 +15%）
- 质变：`ult_overflow_heal`（**溢出治疗 → 敌方扣血**，用户核心提议）

**玩法体验**：
- 对友军放一个"回复 280% ATK + 给血最低友军等量治疗"
- 目标满血后溢出部分转化为对敌方群体的伤害
- 在"全队血线健康"时此技能从回复转为输出

### 5.2 BD-B「DOT 雪球」流

**功法书**：金品神通「九天玄火阵」
- 核心：`core_burn`（灼烧核心）
- 增幅 1：`amp_target_all`（群攻）
- 增幅 2：`amp_duration_long`（+2 回合，灼烧持续 5 回合）
- 触发：`trigger_on_debuff`（每条 debuff 伤害+15%）
- 质变：`ult_dot_detonate`（**DOT 引爆**，敌方 DOT 层数 ×15% 秒伤）

**玩法体验**：
- 第 1 回合：群体灼烧铺底
- 第 2~5 回合：每次攻击都引爆 DOT 造成额外伤害
- 与「万毒归一」被动（DOT+30%）配合指数级雪球

### 5.3 BD-C「护盾反打」流

**功法书**：金品神通「厚土盾」改造
- 核心：`core_shield`（护盾 300% ATK）
- 增幅 1：`amp_duration`（+1 回合）
- 增幅 2：`amp_dmg_small`（护盾量 +30% → 390%）
- 触发：`trigger_low_hp`（残血时护盾量 ×2）
- 质变：`ult_shield_to_dmg`（**护盾消失时转化 50% 伤害还击**）

**玩法体验**：
- 血低 → 爆盾 780% ATK → 敌方打破盾时反被炸 390% 伤害

## 八、实现要点（给开发参考）

### 8.1 数据结构

```typescript
interface SkillBook {
  id: string;                // 功法书 ID
  rarity: Rarity;            // 品质决定孔位数
  element: Element | null;   // 原功法属性
  slots: SkillStoneSlot[];   // 最多 5 个槽位
  slot_limits: {             // 每槽位类型约束
    0: 'core',
    1: 'amp',
    2: 'amp',
    3: 'trigger',
    4: 'ultimate'
  }
}

interface SkillStone {
  id: string;
  type: 'core' | 'amp' | 'trigger' | 'ultimate';
  effects: EffectModifier[];  // 效果修饰符管线
  mutex_tags?: string[];       // 互斥标签（target/multi_hit）
}
```

### 8.2 效果管线（Resolver Pipeline）

结算顺序（严格按此顺序，避免"先乘后加"问题）：

```
1. 读取核心石 → 得到 baseEffect { type, multiplier, debuff, duration }
2. 遍历增幅石 → 修改 baseEffect（倍率+、目标数+、持续+）
3. 应用灵根匹配 bonus（走现有 spiritual_root_synergy 规则）
4. 触发石 → 检查条件，满足则再次修改 baseEffect
5. 质变石 → 后处理回调（溢出转换、吸血、引爆等）
6. 命中判定 / 抗性计算（走现有 debuff_vs_resist）
7. 最终输出
```

### 8.3 UI 需求

- 功法详情页：可视化 5 个孔位 + 实时组合预览（"当前配方：群攻 350% 灼烧 5 回合"）
- 石头库存：按类型分栏，支持拖拽镶嵌
- Build 模板：玩家可收藏配方，一键套用
- 配方推荐：根据灵根自动推荐"最佳核心石"

### 8.4 平衡测试

- 自动化模拟：接入现有 `sim-report-v3.md` 流程，跑 "5 孔满配 vs 同品质传统功法" 战斗
- 红线：DPS 差距 ≤ 30%、生存差距 ≤ 20%
- 监控：记录玩家实际镶嵌组合分布，发现"98% 玩家都用 X+Y 组合"就说明该组合 OP

### B.4 结构性问题（不是护栏问题，但阻塞迁移）

#### 问题 1：被动功法的"复合属性诉求"与"孔位限制"冲突

旧被动常见 3~4 属性复合（如不动如山=DEF+HP+控抗，不灭金身=DEF+减伤+土抗+免死）。
新系统紫品书 4 孔、金品书 5 孔，需要同时承载 3~4 个独立 `amp_*` 属性石。
→ 结果：装得下主要属性，第二/第三属性被迫丢弃。

**修复方向（三选一）**：
- **A) 核心石底座升级**：每个被动核心石自带 2~3 个小加成（如 `core_passive_def` 自带 DEF+15% + 控抗+5%）
- **B) 复合增幅石**：新增 `amp_def_hp_combo`（DEF+8% + HP+5%）等"二合一"石头
- **C) 为旧复合功法定制核心石**：`core_passive_iron_shirt` = DEF+12% + 控抗+10% + 被暴击反弹10%（旧铁布衫整体打包）

> **建议**：P0 迁移阶段走 C 路线（定制化），P2+ 逐步重构为 A+B（模块化）。

#### 问题 2：神通的"主效果 + 自我 buff"双轨

多个天品神通（地裂天崩、时光凝滞、九天玄火阵变种）都有"打伤害的同时给自己 buff"。
新系统的核心石只定义主效果，没有"附加自我 buff"维度。
→ 结果：自我 buff 全部丢失，伤害能对齐但辅助能力衰减。

**修复**：新增 `amp_self_buff_*` 系列增幅石（占用增幅孔），或让"伤害+自 buff"成为**质变石**的一种模式。

#### 问题 3：被动的"被打触发"型效果无映射

旧被动如"荆棘之体被打 10% 中毒"、"焚身火甲被打 10% 灼烧"、"铁布衫被暴击反弹 10%"。
新系统没有"被击反应式"石头。
→ 结果：这些特色被动的"灵魂效果"完全丢失。

**修复**：新增触发类石头（但触发孔已被 `trigger_*` 系列占用）：
- `trigger_on_hit_taken_poison`（被打触发中毒）
- `trigger_on_hit_taken_burn`（被打触发灼烧）
- `trigger_on_crit_taken_reflect`（被暴击反弹）

或更简洁：让核心石承载这些特色（见问题 1 C 方案）。

### B.5 最终建议

1. **数值护栏层面**：🟢 所有 46 映射 **0 突破**，护栏健康
2. **结构层面**：🟡 **9/46（20%）严重偏差**，主要集中在"复合被动"和"神通+自 buff"上
3. **石头库层面**：🟡 需追加 **~31 颗石头**（17 在 A.1 + 14 在 B.3/B.4），总量从 45 → ~76
4. **P0 迁移可行性**：
   - **可行**：25 个完全等价 + 12 个轻微偏差 = 37/46（80%）无痛迁移
   - **需红利补偿**：剩余 9 个严重偏差的功法，给玩家"迁移补偿券"（可兑换任意同品质石头 1 颗 + 50 石灵碎片），**降低玩家不满**
5. **长期方向**：
   - 若走"纯模块化"（石头库精简）：接受 20% 功法小幅弱化，用新 BD 自由度补偿
   - 若走"定制核心石"（石头库膨胀到 ~76）：100% 等价迁移，但石头 UI/运营复杂度上升

### B.6 一句话结论

> **护栏没问题，但"孔位"这个物理限制让部分复合型旧功法天然装不进新体系。要么加新石头（方案 C），要么接受小幅弱化+给玩家红利（方案 B），不存在"既保护栏又零改动"的路径。**
