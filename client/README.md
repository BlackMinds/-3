# 万界仙途 - 前端客户端

放置修仙RPG网页游戏前端,基于 Vue 3 + TypeScript + Vite 构建。

## 技术栈

- **框架**: Vue 3 (Composition API + `<script setup>`)
- **构建工具**: Vite 5
- **语言**: TypeScript
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP**: Axios

## 项目结构

```
src/
├── api/                    # API 请求层
│   ├── request.ts          # Axios 实例(baseURL + JWT 拦截器)
│   ├── auth.ts             # 登录/注册
│   ├── character.ts        # 角色/头像
│   ├── game.ts             # 游戏数据/保存/更新
│   └── skill.ts            # 功法
├── game/                   # 游戏数据定义(纯 TS)
│   ├── types.ts            # 类型定义
│   ├── data.ts             # 静态数据(25张地图T1-T10、8大境界、灵根、五行)
│   ├── battleEngine.ts     # 前端战斗引擎(保留类型定义,战斗已迁移后端)
│   ├── skillData.ts        # 功法数据(6主修+21神通+19被动=46个)
│   ├── equipData.ts        # 装备系统(7槽位、4武器类型、15种副属性、强化公式)
│   ├── pillData.ts         # 丹药配方(5战斗丹+4突破丹)
│   ├── herbData.ts         # 灵草分级(7种x6品质)
│   └── caveData.ts         # 洞府建筑(7座,升级费用/产出公式)
├── stores/
│   ├── user.ts             # 登录状态(JWT token)
│   └── game.ts             # 游戏状态(战斗调度/角色/地图/离线)
├── views/
│   ├── Login.vue           # 登录页(水墨风)
│   ├── CreateCharacter.vue # 角色创建(灵根选择+道号)
│   └── Game.vue            # 游戏主界面(5标签页+弹窗系统+Toast提示)
├── router/index.ts         # 路由(JWT守卫)
├── main.ts
├── style.css               # 水墨暗色主题
└── App.vue
```

## 核心功能

| 模块 | 说明 |
|------|------|
| 登录/注册 | JWT 鉴权,水墨风登录页 |
| 角色创建 | 五行灵根选择 + 道号,头像上传 |
| 等级系统 | Lv.1~200,打怪自动升级,按段递增属性(1-50/51-100/101-150/151-200) |
| 境界系统 | 练气→筑基→金丹→元婴→化神→渡劫→大乘→飞升(8大境界),手动突破 |
| 地图历练 | 25张地图(T1~T10),每波1-5只怪,1%概率Boss |
| 战斗系统 | 服务端计算,前端展示日志,防作弊 |
| 功法系统 | 46个功法(1主修+3神通+3被动),Lv5上限,每级+15%效果 |
| 装备系统 | 7槽位,4种武器(剑/刀/枪/扇),6品质,15种副属性,强化+10,等级限制 |
| 炼丹系统 | 灵草品质系数(1.0x~5.0x),失败全扣,时间制buff(1-8小时) |
| 洞府系统 | 7座建筑(聚灵阵/灵田/聚宝盆/演武堂/藏经阁/炼丹房/炼器房) |
| 离线挂机 | 开始离线→结束并领取,最长12小时,70%效率,掉落装备/功法/灵草 |
| Toast提示 | 全局操作反馈(成功绿色/失败红色/信息金色) |

## 5 个标签页

1. **历练** — 选地图,开始历练/离线挂机,战斗日志,统计面板,掉落表
2. **角色** — 等级/属性/抗性/元素强化面板 + 装备槽位穿戴/强化/背包管理
3. **炼丹** — 灵草库存,选品质炼丹,丹药背包使用,buff状态显示
4. **功法** — 已装备功法(1主修+3神通+3被动) + 背包残页,升级,详情悬浮
5. **洞府** — 建筑升级进度,灵田种植/收获,聚灵阵/聚宝盆产出领取

## 战斗数据流(后端计算)

```
前端: 点击开始历练 → store.startBattle()
  → 每次战斗调用 POST /api/battle/fight
  → 后端: 读取角色属性+装备+技能+buff+洞府
  → 后端: 生成1-5只怪物,完整战斗引擎计算
  → 后端: 经验/灵石/等级/掉落直接入库
  → 后端: 返回战斗日志 + 最新角色数据
  → 前端: 逐条播放日志(每秒一条) + 同步血条
  → 前端: 日志播完 → 更新本地显示 → 下一场
```

## 离线挂机流程

```
前端: 点击"开始离线" → POST /api/game/offline-start
  → 后端: 记录 offline_start 时间
前端: 点击"结束离线" → GET /api/game/offline-status (预览)
  → 弹窗显示预估收益(红字: "点击结算后发放")
  → 点击"结束离线并领取" → POST /api/game/offline-claim
  → 后端: 计算离线时长(上限12h) × 地图怪物收益 × 70%效率
  → 后端: 入库经验/灵石 + 生成装备/功法/灵草 + 检查升级
  → 前端: 显示结算结果,刷新背包
```

## 弹窗系统

| 弹窗 | 触发方式 |
|------|----------|
| 掉落表 | 顶栏"掉落表"按钮,显示每张地图怪物/装备品质/功法/灵草 |
| 帮助文档 | 顶栏"帮助"按钮,完整游戏机制说明 |
| 统计 | 历练中"统计"按钮,击杀/修为/灵石/每分钟效率 |
| 境界突破 | 角色面板突破按钮,修为够时可突破 |
| 装备强化 | 点击已穿戴装备,+10上限/+6起有失败 |
| 离线收益 | "结束离线"按钮,预览→领取 |
| 灵田帮助 | 洞府灵田"?"按钮,种植说明 |

## 启动

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # 生产构建
npm run preview    # 预览生产构建
```

## 环境要求
- Node.js >= 18
- 后端服务运行在 http://localhost:3001
